# ReminderBot — Контекст проєкту

## Опис проєкту
Telegram-бот для створення, керування та відстеження нагадувань з підтримкою інтервалів, разових та повторюваних завдань, збереженням у SQLite (aiosqlite) та розкладом через APScheduler.

## Поточний стан
- **Поточна гілка Git:** `feature/rich-ui`
- **Основні бібліотеки:** `python-telegram-bot==22.5`, `APScheduler==3.11.0`, `aiosqlite==0.19.0`
- **Реалізовано:** 
  1. Повний редизайн інтерфейсу на Telegram Bot API MarkdownV2 з безпечним екрануванням, Rich Visual Cards, майстром створення з прогрес-баром та інтерактивною дашборд-пагінацією.
  2. Виправлено баг створення та завершення одноразових завдань із кількома часами (наприклад `16:30`, `17:30`). Тепер завдання планує джоби для кожного часу і не видаляється до тих пір, поки не відбудуться або не будуть виконані всі заплановані часи.

## Що зроблено (Останні зміни)
- **`core/scheduler.py` (`_schedule_one_time_task`):**
  - Виправлено ітерацію по всьому списку `times` при створенні одноразових завдань на конкретну дату або день тижня.
  - Додано метод `has_remaining_jobs` для перевірки залишкових активних робіт завдань.
- **`bot/handlers.py` (`_handle_done`):**
  - Одноразове завдання більше не видаляється миттєво при відмітці першого часу, якщо для нього є майбутні заплановані нагадування.
- **Новий модуль `bot/ui_helpers.py`:**
  - Реалізовано функцію `escape_md` для захисту від крашів при вводи спецсимволів (`*`, `_`, `(`, `)` тощо).
  - Форматування карточок задач `format_task_card`, сповіщень `format_reminder_notification` та прогрес-барів `format_progress_header`.

## Наступні кроки
- Перезапустити тестового бота з новими виправленнями та перевірити роботу.
- Злити гілку `feature/rich-ui` в `main`.
